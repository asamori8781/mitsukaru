# ミツカル(Phase 0 試作版)

PC内に散在するファイルを、曖昧なキーワードから探し出すWindows向けデスクトップツールです。

生成AIは「検索キーワードの展開」にのみ使用し、ファイルの走査・検索はすべてローカルで完結します。
**Phase 0ではファイルの中身は一切読みません。** ファイル名・パス・拡張子・サイズ・更新日時のみを
DBに登録し、ファイル名に手がかりがない検索がヒットしないのは仕様上許容しています。
全文インデックス・埋め込み・ファイルプレビューはPhase 1のスコープです。

## 構成

```
config.py     設定(data/config.json)の読み書き・既定値・データ配置パス解決
db.py         SQLiteスキーマ(files / files_fts)の初期化・バージョンチェック
indexer.py    フォルダ走査・差分スキャン・進捗管理・Phase1予測サイズ算出
ai_client.py  AI API呼び出しの隔離(キーワード展開・接続テスト)。モックモード対応
search.py     FTS5(trigram)によるローカル検索・ランキング
server.py     FastAPIルーティング・利用ログ記録・静的ファイル配信
main.py       エントリポイント(pywebviewウィンドウ+FastAPIバックグラウンド起動)
static/       フロントエンド(HTML/CSS/JS、フレームワーク不使用)
data/         実行時生成、git管理外(config.json, mitsukaru.db)
logs/         実行時生成、git管理外(api_log.jsonl, usage_log.csv)
```

## 開発環境での起動手順(uvicorn直起動)

pywebviewのウィンドウを使わず、通常のブラウザで動作確認する場合は以下の手順です
(Windows以外の開発機でのフロー確認にも使えます)。

```bash
python -m venv venv
venv\Scripts\activate        # Windowsの場合。macOS/Linuxは source venv/bin/activate
pip install -r requirements.txt
uvicorn server:app --reload --port 8000
```

ブラウザで `http://127.0.0.1:8000/` を開くとUIが表示されます。

実際にpywebviewウィンドウとして起動する場合は以下です(Windows想定)。

```bash
python main.py
```

## 初回設定の流れ

1. `data/config.json` が存在しない場合、起動時にAPI初期設定モーダルが表示されます。
   - ベースURL・APIキー・モデル名・タイムアウト秒(既定20秒)を入力し、「接続テスト」で疎通確認できます。
   - 「スキップ」を選ぶとAPIを使わない**モックモード**で起動し、画面上部に
     「モックモード動作中(API未設定)」と常時表示されます。モックモード・API接続失敗時は、
     入力文を空白で分割した語をそのままキーワードとしてローカル検索にフォールバックします
     (日本語は分かち書きされないため、文中に空白がない場合は文全体が1語として扱われ、
     ヒットしないことがあります。これは仕様上の制約です)。
2. 続けて対象フォルダ選択画面が表示されます。「このPCの全ドライブをスキャン」または
   個別フォルダの追加(フォルダ選択ダイアログ、ブラウザ直起動時はパス直接入力)を選び、
   スキャンを開始します。
3. スキャン完了後、登録件数・DBファイルサイズ・Phase 1予測サイズ(概算)が表示され、
   検索画面に進めます。

以降の起動では、アプリ起動時と検索実行時(前回差分スキャンから設定分数以上経過している場合)に
バックグラウンドで自動差分スキャンが走るため、手動メンテナンスは不要です。

## PyInstallerでのビルド手順(onedir)

```bash
pip install pyinstaller
pyinstaller --name mitsukaru --onedir --windowed --add-data "static;static" main.py
```

- macOS/Linuxでビルドする場合は `--add-data "static:static"`(区切り文字が `:`)にしてください。
- 生成物は `dist/mitsukaru/` 配下に展開されます。`data/` と `logs/` は実行ファイルと同階層に
  自動生成されるため、フォルダごとコピーすればポータブルに動作します。
- **想定サイズ目安**: FastAPI/Starlette/Uvicorn/pywebview/requests一式とPythonランタイムを
  同梱した場合、onedir全体でおおよそ数十〜100MB程度になる見込みです(pandas/numpyは
  未使用のため肥大化は抑えられます)。実測値は開発機でのビルド後に確認してください。
- Windowsではpywebviewの既定バックエンド(WebView2)を利用するため、WebView2 Runtime
  (Windows 10/11には標準搭載されていることが多い)が必要です。

## APIキーの取り扱いに関する注意

- `data/config.json` にAPIキーを**平文で保存**します。`.gitignore` で `data/` を
  リポジトリ管理外にしていますが、ファイル自体はローカルディスクに平文で残るため、
  共有PCや会社貸与PCでの取り扱い、バックアップ・持ち出し時には十分注意してください。
- APIへ送信するのはユーザーの検索文と固定の展開指示プロンプトのみです。ファイル名・パス・
  ファイルの内容・スキャン結果は一切送信しません(送信処理は `ai_client.py` に隔離しています)。
- 送受信内容は `logs/api_log.jsonl` にローカル保存されます(透明性ログ、APIキーは記録しません)。
- 別のPCや業務環境へ展開する際は、`data/config.json` を差し替えるだけで接続先を
  切り替えられます(接続情報はコードにハードコードしていません)。

## Phase 1で追加予定の機能(Phase 0のスコープ外)

- ファイルの全文インデックス(テキスト/Office/PDF等の内容抽出・索引化)
- 各種フォーマット向けの抽出器(docx/xlsx/pptx/pdf 等)
- 実ファイルからのテキスト抽出量に基づくインデックスサイズの実測・見積もり精度向上
- 埋め込み(ベクトル検索)による意味的な検索
- ファイルプレビュー

- [ ] 対象フォルダ選択で、任意のユーザーフォルダ(例: ドキュメントフォルダ)を追加してスキャン開始

- [ ] スキャン完了時に登録件数・DBファイルサイズ・Phase 1予測サイズが表示される

- [ ] `logs/usage_log.csv` に検索日時・入力クエリ・展開語数・ヒット件数・開いたファイルの有無が記録される
- [ ] 設定画面でDB統計・対象フォルダ・除外設定・API設定を確認・変更できる
